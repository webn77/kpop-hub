# 업무 일기 — 2026-05-29

오늘은 KPOP HUB 커뮤니티를 Render에 배포하기 위한 첫 단계를 진행했다.

원래 지난주(05-22)에 Railway 배포 설정이 들어가 있었는데, 수업은 Render 기준이라 방향을 Render로 맞췄다. 다행히 Railway 작업 때 whitenoise·STATIC·SECRET_KEY 환경변수가 이미 들어가 있어서 Render 가이드의 절반은 이미 만족된 상태였다. 그래서 중복 작업 없이 빠진 부분만 채웠다.

가장 신경 쓴 부분은 DATABASES 설정이다. 가이드는 dj_database_url로 갈아끼우라고 했지만, 기존 로컬 SQLite에 WAL 최적화 PRAGMA가 걸려 있어서 그냥 덮으면 로컬 성능 설정이 사라진다. 그래서 DATABASE_URL 환경변수 유무로 분기해서 — 운영은 PostgreSQL, 로컬은 기존 SQLite(WAL 보존) — 양쪽 다 살렸다.

수정 후 manage.py check로 로컬이 안 깨지는지 확인했고, DB 엔진이 여전히 sqlite3, DEBUG=True인 것까지 검증했다. 운영 설정이 로컬 개발을 방해하지 않는다는 걸 눈으로 확인하니 안심이 됐다.

다음은 render.yaml(2/4)을 만들 차례다. Railway 잔재 파일을 정리할지도 그때 같이 판단하자.
