# 업무 일기 — 2026-05-15

오늘 11주차 수업 주제로 Django로 KPOP HUB 커뮤니티를 시작했다.

## 한 일

Phase 0 — 뼈대 세팅 완료. 기존에 docs/ 폴더에 PRD, TCD, 기능명세서 등 문서를 먼저 작성해두고 코드를 시작하는 방식이 맞다는 걸 다시 확인했다. 문서가 있으니 settings.py를 짤 때 어떤 앱이 필요한지, allauth 설정이 왜 필요한지 바로 보였다.

커스텀 User 모델(AbstractUser)을 처음부터 만들어두는 것이 중요하다. 나중에 추가하면 migration이 꼬여서 고생한다. 이번에는 계획대로 accounts/models.py를 먼저 작성하고 makemigrations → migrate 순서로 진행해서 깔끔하게 됐다.

whitenoise를 처음부터 넣어둔 것도 잘한 결정이다. Railway에 배포할 때 정적 파일 때문에 admin CSS가 깨지는 문제를 미리 막았다.

## 배운 점

- AUTH_USER_MODEL을 settings에 넣으면 반드시 해당 모델이 먼저 있어야 migrate 가능
- SQLite WAL 모드: 쓰기 성능 개선 + 동시 읽기 허용 (PRAGMA journal_mode=WAL)
- whitenoise: Django 내장 정적파일 서버 대신 사용. production에서 필수
