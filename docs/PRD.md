# PRD.md — K-POP 팬 커뮤니티 프로젝트 요구사항 정의서

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | KPOP HUB |
| 목적 | K-POP 팬들이 글이 묻히지 않고, 가볍게, 오래 소통할 수 있는 커뮤니티 |
| 타겟 | K-POP 팬 (10~30대, 모바일 중심) |
| 기술스택 | Django 5.x + SQLite + Tailwind CSS + HTMX |

## 2. 기존 서비스 문제점 및 해결

| 서비스 | 문제 | 우리 해결책 |
|---|---|---|
| 카카오톡 오픈채팅 | 글이 금방 묻혀버림 | 게시판 구조로 글 보존 |
| 인스타그램 | 사용자끼리 소통 어려움 | 댓글·대댓글·좋아요 시스템 |
| 네이버 카페 | 너무 복잡하고 무거움 | 심플한 단일 페이지 구조 |
| 스레드 | 정보가 너무 빨리 흘러감 | 카테고리별 게시판으로 정보 보존 |

## 3. 핵심 기능 요구사항

### P0 — 필수 (MVP)

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 회원가입/로그인 | 이메일+비밀번호, 세션 관리, 프로필 표시 | P0 |
| 공지사항 | 관리자만 작성, 상단 고정 가능 | P0 |
| 소통 공간 | 자유게시판, 좋아요, 조회수, 댓글, 대댓글, 이미지 업로드 | P0 |
| 관리자 대시보드 | 회원 관리, 게시글 삭제, 공지 작성 권한 | P0 |

### P1 — 중요

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 주요 일정 | 캘린더 기반 K-POP 일정 등록/조회 | P1 |
| 투표 | 단일/다중 선택, 설문조사로 확장 가능 | P1 |

### P2 — 선택

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 소셜 로그인 | 구글 로그인 연동 | P2 |
| 알림 | 댓글 알림 | P2 |

## 4. 성공 지표

- 회원가입 후 게시글 작성까지 3분 이내
- 모바일 페이지 로딩 2초 이내
- 관리자 페이지에서 회원/게시글 관리 가능
