# TR.md — Test Report

## 테스트 철학

바이브 코딩 환경에서 테스트는 **기능 동작 확인** 중심으로 진행한다.
복잡한 자동화 테스트보다 **브라우저에서 직접 확인**을 우선한다.

## 앱별 커버리지 목표

| 앱 | 핵심 테스트 항목 | 목표 |
|---|---|---|
| accounts | 회원가입, 로그인, 로그아웃, 권한 체크 | 브라우저 확인 |
| notices | 공지사항 작성(관리자), 목록/상세 조회 | 브라우저 확인 |
| posts | 게시글 CRUD, 좋아요, 댓글, 이미지 업로드 | 브라우저 확인 |
| events | 일정 등록, 캘린더 조회 | 브라우저 확인 |
| polls | 투표 생성, 참여, 결과 확인 | 브라우저 확인 |

## 기능별 테스트 시나리오

### 인증 시스템
- [ ] 신규 회원가입 → 로그인 성공
- [ ] 잘못된 비밀번호 → 에러 메시지 표시
- [ ] 로그아웃 → 보호된 페이지 접근 차단
- [ ] 관리자 계정 → 관리자 페이지 접근 가능

### 소통 공간 (posts)
- [ ] 로그인 상태에서 게시글 작성
- [ ] 이미지 포함 게시글 업로드
- [ ] 좋아요 버튼 클릭 → 숫자 변경 (페이지 새로고침 없이)
- [ ] 댓글 작성 → 목록에 즉시 표시
- [ ] 대댓글 작성 → 댓글 하위에 표시
- [ ] 본인 게시글만 수정/삭제 가능

### 관리자
- [ ] Django Admin 접속 (http://127.0.0.1:8000/admin)
- [ ] 회원 목록 조회
- [ ] 게시글 삭제
- [ ] 공지사항 작성

## 보안 테스트 시나리오

- [ ] 비로그인 상태에서 게시글 작성 시도 → 로그인 페이지로 리다이렉트
- [ ] 타인 게시글 수정 URL 직접 접근 → 403 에러
- [ ] 관리자 페이지 일반 유저 접근 → 403 에러
